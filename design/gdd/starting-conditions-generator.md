# Starting Conditions Generator

> **Status**: Designed
> **Author**: User + game-designer, systems-designer
> **Last Updated**: 2026-03-22
> **Implements Pillar**: Life is Unfair (and That's the Point)

## Overview

The Starting Conditions Generator produces the randomized birth circumstances for
each new life: family background, socioeconomic class, geographic location, family
structure, initial traits, and starting relationships. This is the "hand you're
dealt" — the foundation of the "Life is Unfair" pillar. No two births should feel
identical, and the starting conditions should meaningfully shape the trajectory of
the entire life.

The system uses **combinatorial generation**: individual components (region, class,
family structure, parent traits) are rolled independently and combined, creating
high variety from a manageable number of authored components. Each component
influences starting attribute values, available event pools, and the tone of early
life events.

## Player Fantasy

The moment of birth should feel like opening a mystery box — "Who am I? Where am
I? What have I been given?" The player should immediately feel the weight of their
circumstances: a wealthy family in a stable country feels different from a poor
family in a struggling region. This isn't about judgment — it's about perspective.
The "Life is Unfair" pillar means some starts are objectively harder, and that
contrast is what drives empathy across playthroughs.

## Detailed Design

### Core Rules

#### Generation Components

Birth conditions are generated by rolling 6 independent components:

| Component | Description | Variety Count | Affects |
|-----------|------------|---------------|---------|
| **Region** | Geographic/cultural context | 8 regions | Name pool, cultural events, location identity |
| **Socioeconomic Class** | Starting wealth/opportunity | 4 classes | Attribute baselines, event eligibility, education access |
| **Family Structure** | Who raises you | 6 types | Starting relationships, family events, stability |
| **Parent Personality** | Parents' dominant traits | Random | Personality spectrum seeds, parenting events |
| **Health Lottery** | Starting health circumstances | Weighted random | Health attribute baseline, potential chronic conditions |
| **Era Modifier** | When you're born (optional/future) | 1 (modern default) | Contextual flavor, technology access |

#### Region Definitions

Regions are cultural/geographic archetypes, not real countries. This avoids
stereotyping while capturing meaningful lifestyle differences.

| Region | Flavor | Name Pool | Event Color | Starting Wealth Modifier |
|--------|--------|-----------|-------------|------------------------|
| North American Suburban | Suburban sprawl, car culture, school system | English names | Stable, routine | +10 |
| European Urban | Dense city, public transit, cultural access | European names | Cultured, fast-paced | +5 |
| East Asian Metro | High-density, education pressure, technology | East Asian names | Competitive, structured | +5 |
| Latin American | Family-centric, community, economic contrast | Latin names | Warm, resilient | -5 |
| South Asian | Extended family, tradition, education focus | South Asian names | Traditional, aspirational | -5 |
| African Urban | Growing city, community bonds, opportunity gaps | African names | Resourceful, community | -10 |
| Rural/Small Town | Any region, small community, nature access | Region-appropriate | Slow, close-knit | -5 |
| Immigrant Experience | Any origin region, new country adaptation | Mixed/origin names | Adaptive, dual-identity | -10 |

**Rules**:
1. Regions are **not stereotypes** — they provide starting context and name pools,
   not deterministic life paths. A child from any region can reach any outcome.
2. Wealth modifiers are starting adjustments only. They don't cap growth.
3. The name pool includes 50+ names per region with cultural variety.
4. Events reference region via tags, not hardcoded region checks. This allows
   events to work across multiple regions that share characteristics.

#### Socioeconomic Class

| Class | Wealth Start | Education Start | Stress Start | Description |
|-------|-------------|-----------------|-------------|-------------|
| Upper | 75-85 | 60-70 | 20-30 | Financial security, high expectations, access to opportunity |
| Middle | 45-55 | 45-55 | 30-40 | Comfortable but not wealthy, standard education path, stable |
| Working | 25-35 | 30-40 | 40-50 | Tight finances, practical focus, some opportunity barriers |
| Lower | 10-20 | 15-25 | 50-65 | Financial hardship, limited access, high stress from scarcity |

**Rules**:
1. Class is rolled with weighted probability: Upper 10%, Middle 40%, Working 35%, Lower 15%.
   (Roughly reflects global distribution in developed economies.)
2. Starting attribute values are randomized within the ranges above.
3. Class affects event eligibility (e.g., "private school" events require Upper/Middle)
   but never locks out major life outcomes.

#### Family Structure

| Structure | Starting Relationships | Stability Modifier | Description |
|-----------|----------------------|-------------------|-------------|
| Two-parent stable | Mother + Father (closeness 70-90) | +10 Happiness | Traditional nuclear family, both parents present |
| Two-parent strained | Mother + Father (closeness 30-60) | -5 Happiness, +10 Stress | Both parents present but tension in the household |
| Single parent | One parent (closeness 60-80) | +5 Stress | One dedicated parent, other absent or minimal |
| Extended family | Parent + Grandparent + Aunt/Uncle | +15 Relationships | Multi-generational household, strong family network |
| Blended family | Parent + Step-parent + Step-sibling | Varies | Complex family dynamics, adaptation challenges |
| Foster/Institutional | Caregiver (closeness 20-40) | -15 Happiness, +20 Stress | No biological family present, institutional care |

**Rules**:
1. Family structure is rolled with weighted probability: Two-parent stable 35%,
   Two-parent strained 20%, Single parent 20%, Extended 10%, Blended 10%, Foster 5%.
2. Each structure creates 1-4 starting relationships with appropriate types and closeness.
3. Siblings are rolled separately: 0 siblings (30%), 1 sibling (35%), 2 siblings (25%), 3+ (10%).

#### Parent Personality Generation

Each parent gets 2-3 personality traits randomly assigned from a pool:

**Positive traits**: supportive, encouraging, hardworking, creative, patient, funny, wise
**Negative traits**: demanding, absent, strict, unpredictable, critical, anxious, controlling
**Neutral traits**: quiet, traditional, ambitious, religious, practical, free-spirited

**Trait influence on spectrums**: Each parent trait contributes ±5 to a single
relevant spectrum. Both parents' influences stack. Total parent seed is clamped to
±15 before random variance. Example: Two parents both with "supportive" trait →
child Empathy spectrum seeded at +10 before random variance of ±5.

**Rules**:
1. Each parent gets 1-2 positive and 0-1 negative traits (weighted: most parents are
   mixed, not purely good or bad).
2. Parent traits influence early-life events: a "demanding" parent generates pressure
   events; a "supportive" parent generates encouragement events.
3. Parent traits seed the child's personality spectrums: a "creative" parent gives
   the child a small starting nudge on related spectrums (±5-10 range, not deterministic).

#### Health Lottery

| Roll | Starting Health | Condition | Probability |
|------|----------------|-----------|-------------|
| Excellent | 85-95 | None | 40% |
| Good | 70-84 | None | 35% |
| Fair | 50-69 | Minor (allergies, nearsightedness) | 15% |
| Chronic | 30-49 | Chronic condition (asthma, disability) | 8% |
| Severe | 15-29 | Serious condition (rare disease, disability) | 2% |

**Rules**:
1. Health lottery tags are added: "chronic_asthma", "visual_impairment", etc.
2. Chronic and severe conditions unlock specific event chains and modify event
   eligibility throughout life.
3. Conditions are NOT death sentences — they're life modifiers. A character with
   chronic asthma can still live a full, rich life.

#### Complete Generation Process

```
1. Roll Region (weighted random)
2. Roll Socioeconomic Class (weighted random)
3. Roll Family Structure (weighted random)
4. Generate Parent(s) — names from region pool, personality traits
5. Generate Sibling(s) — if any, names from region pool
6. Roll Health Lottery (weighted random)
7. Calculate starting attributes:
   - Health: from Health Lottery
   - Wealth: Class baseline + Region modifier + random(±5)
   - Education: Class baseline + random(±5)
   - Career: 0 (infant)
   - Relationships: from family closeness average
   - Happiness: 50 + Family Structure modifier + random(±10)
   - Stress: Class baseline + Family Structure modifier + random(±5)
8. Set personality spectrums: 0 (neutral) + parent trait seeds (±5-10)
9. Set initial tags: region tag, class tag, family structure tag, any health tags
10. Set initial flags: derived from above
11. Create Character State Model with all layers populated
```

**Combinatorial variety**: 8 regions × 4 classes × 6 family types × health
variations × parent trait combinations = **thousands** of unique starting
conditions from ~30 authored components.

### States and Transitions

The Starting Conditions Generator is stateless — it runs once per life and produces
output. No internal state machine.

| Trigger | Action | Output |
|---------|--------|--------|
| "New Life" from Title | Full random generation | Complete CharacterState |
| "New Life" from Tree | Full random generation (independent of previous life) | Complete CharacterState |
| "Continue Legacy" from Tree | Generational generation (reads parent state) | Complete CharacterState modified by inheritance |

**Note**: "Continue Legacy" delegates to the Generational Inheritance System, which
calls back to this generator with inheritance overrides.

### Interactions with Other Systems

| System | Direction | Interface |
|--------|-----------|-----------|
| **Character State Model** | Writes to | `generateCharacter() → CharacterState` — creates a fully populated state object |
| **Life Stage Definitions** | Reads from | Reads Infancy stage definition to set initial life phase |
| **Generational Inheritance System** | Called by (future) | `generateCharacter(inheritanceOverrides) → CharacterState` — accepts partial overrides for inherited traits |
| **Timeline Engine** | Triggers | After generation, signals the Timeline Engine to begin the first year tick |
| **Event Content Database** | Influences | Starting tags and conditions determine which events become eligible from birth |

## Formulas

### Starting Attribute Calculation

```
attribute_value = class_baseline + region_modifier + structure_modifier + random_variance
attribute_value = clamp(attribute_value, 0, 100)
```

| Variable | Type | Range | Source | Description |
|----------|------|-------|--------|-------------|
| class_baseline | int | 10-85 | Class definition | Base value for this attribute from socioeconomic class |
| region_modifier | int | -10 to +10 | Region definition | Regional adjustment |
| structure_modifier | int | -15 to +20 | Family structure | Family stability impact |
| random_variance | int | -10 to +10 | Random roll | Per-attribute randomization |

### Parent Trait Personality Seeding

```
spectrum_seed = sum(trait_influences) + random(-5, +5)
spectrum_seed = clamp(spectrum_seed, -15, +15)
```

Each parent trait has a small influence on relevant spectrums:
- "creative" parent → child Conformity spectrum: -5 (slightly rebellious)
- "strict" parent → child Conformity spectrum: +5 (slightly conformist)
- "encouraging" parent → child Courage spectrum: +5 (slightly brave)

Seeds are small (±15 max) to leave room for player-driven personality development.

## Edge Cases

| Scenario | Expected Behavior | Rationale |
|----------|------------------|-----------|
| All attributes roll at minimum | Ensure at least one attribute ≥ 30 by boosting the lowest | Even the hardest start should have one bright spot — total despair isn't fun |
| Both parents roll only negative traits | Cap negative traits at 1 per parent. Re-roll if both are negative-only. | Completely toxic parents with zero positive traits feel cartoonish, not realistic |
| Foster care + Lower class + Severe health | Valid but rare combination (~0.01% chance). These lives are the hardest and the most powerful for empathy | "Life is Unfair" pillar at maximum expression. The game should handle it gracefully |
| Generational inheritance overrides all random rolls | Inheritance provides overrides, not replacements. Random variance still applies on top | Even inherited lives should feel unique, not deterministic copies |
| Player gets same starting conditions twice | Extremely unlikely with combinatorial variety, but possible. Parent names and minor details will still differ | The random variance ensures no two lives are truly identical even with same archetype |
| CSM spectrum initialization alignment | Spectrums are set to parent seed values (±15 max), not zero. Character State Model's "starts near neutral" language means pre-player-choice, not literally zero. | Reconciles SCG seeding with CSM documentation — both docs now aligned. |

## Acceptance Criteria

- [ ] Generator produces a valid CharacterState with all 6 layers populated
- [ ] All attributes are within valid ranges (0-100) after generation
- [ ] Personality spectrums start within ±15 of neutral (0)
- [ ] Family relationships are created matching the rolled family structure
- [ ] Names are culturally appropriate for the rolled region
- [ ] No two consecutive generations produce identical starting conditions
- [ ] Class distribution over 1000 generations matches configured weights (±5%)
- [ ] Health lottery distribution matches configured weights
- [ ] At least one attribute ≥ 30 in every generated character
- [ ] Starting tags correctly reflect region, class, family, and health rolls
- [ ] Generation completes in under 50ms
- [ ] All generation weights and ranges are externalized, not hardcoded
